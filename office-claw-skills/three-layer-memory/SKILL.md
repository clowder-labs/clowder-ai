# 三层记忆法 - OpenClaw Agent 持久化记忆管理

## 📝 技能简介

为 OpenClaw Agent 提供一套完整、经过实战验证的**三层持久化记忆管理方案**，彻底解决会话重启、子代理边界、定时任务隔离导致的记忆断裂问题。

这套方案就是我们现在每天在用的工作流，经过实际使用验证，稳定可靠，开箱即用。

## ✨ 核心特性

- **三层架构**：日常记录 → 长期记忆 → 定期维护，分工清晰
- **文件唯一真相源**：完全符合 Context Relay 理念，不依赖会话内存
- **开箱即用**：直接按照目录结构创建文件就能用
- **低门槛**：不需要复杂配置，符合 OpenClaw 默认规范

## 🏗️ 三层架构

### 第一层：每日日常记录 (`memory/YYYY-MM-DD.md`)
- **用途**：记录当天发生的所有原始内容，流水账式记录
- **位置**：`memory/` 目录下按日期命名
- **内容**：对话要点、决策记录、错误日志、待办事项
- **特点**：原始、完整、不筛选

### 第二层：长期精选记忆 (`MEMORY.md`)
- **用途**：存放最重要、需要长期记住的内容
- **位置**：工作区根目录
- **内容**：重要决策、核心配置、用户偏好、经验教训
- **特点**：精炼、结构化、容易快速检索

### 第三层：心跳自动维护 (`HEARTBEAT.md` + 定期检查)
- **用途**：定期自动整理记忆，保持记忆鲜活
- **机制**：每 4 小时执行一次心跳检查
- **工作内容**：
  1. 阅读最近几天的日常记录
  2. 将重要内容提炼到 `MEMORY.md`
  3. 清理过时信息
  4. 更新状态

## 📁 目录结构

```
workspace/
├── memory/                    # 第一层：日常记录
│   ├── 2026-03-18.md
│   ├── 2026-03-17.md
│   └── heartbeat-state.json  # 心跳状态记录
├── MEMORY.md                 # 第二层：长期精选记忆
├── HEARTBEAT.md              # 第三层：心跳检查清单
├── AGENTS.md                 # 工作区整体规则
├── SOUL.md                   # 安全规则/角色设定
├── IDENTITY.md               # Agent 身份信息
└── USER.md                   # 用户信息
```

## 🎯 使用方法

### 1. 初始化（复制模板）
直接按照上面的目录结构创建文件即可，OpenClaw 默认已经包含这个结构。

### 2. 日常使用
每次对话结束，将重要内容记录到当天的 `memory/YYYY-MM-DD.md`。

### 3. 回答问题前必须检索
在回答任何关于历史、决策、偏好的问题前，必须：
```bash
1. 调用 memory_search 语义搜索 MEMORY.md + memory/*.md
2. 使用 memory_get 获取具体内容
3. 基于文件内容回答，不要依赖记忆
```

### 4. 心跳维护配置
在 `HEARTBEAT.md` 中配置定期检查清单，包含记忆维护步骤：
```markdown
## 检查清单

### 4. 记忆维护
- [ ] 压缩记忆（如果需要）
- [ ] 更新心跳状态到 `memory/heartbeat-state.json`
```

## 💡 最佳实践

1. **多写多整理**：好记性不如烂笔头，任何重要事情都写下来
2. **定期提炼**：心跳检查时把日常记录中重要内容移动到 MEMORY.md
3. **文件优先**：永远相信文件，不相信会话内存，重启也不会丢
4. **精简长期记忆**：MEMORY.md 只放最重要的，保持短小精悍方便检索
