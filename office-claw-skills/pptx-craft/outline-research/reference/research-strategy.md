# 两阶段研究策略详解

本文档是 outline-research 技能的研究策略参考，详述广度检索和深度抓取的执行细节。

---

## 1. 搜索查询构造

### 1.1 查询语言策略

| 主题语言 | 查询策略 | 原因 |
|----------|----------|------|
| 中文主题 | 中英双语查询 | 英文来源通常信息密度更高，中文来源更贴近本土市场 |
| 英文主题 | 英文为主 | 直接匹配目标来源 |
| 行业主题 | 添加行业术语 | 提升检索精准度 |

### 1.2 八路线查询分解

在开展搜索之前，将研究问题分解为 8 条相互独立的搜索路线：

| 路线 | 方向 | 查询模板 |
|------|------|----------|
| 1 | 核心主题 | "{topic} 综述 概览 {year}" |
| 2 | 技术细节 | "{topic} 技术架构 关键参数" |
| 3 | 近期动态 | "{topic} 2025 2026 最新" |
| 4 | 数据统计 | "{topic} 市场规模 统计" |
| 5 | 案例实证 | "{topic} 企业案例 落地" |
| 6 | 对比分析 | "{topic} 对比 竞品" |
| 7 | 权威来源 | "Gartner {topic} report" |
| 8 | 挑战局限 | "{topic} 风险 问题" |

### 1.3 查询模板

**趋势类页面**：
```
"{topic} trend {year}" / "{topic} forecast {year}"
"{主题} 趋势 {年份}" / "{主题} 发展报告"
```

**数据类页面**：
```
"{topic} market size statistics {year}"
"{topic} adoption rate report"
"{主题} 市场规模 {年份}" / "{主题} 统计数据"
```

**技术类页面**：
```
"{topic} technology breakthrough {year}"
"{topic} technical architecture"
"{主题} 技术突破" / "{主题} 架构演进"
```

**案例类页面**：
```
"{topic} case study enterprise"
"{topic} implementation success story"
"{主题} 企业案例" / "{主题} 落地实践"
```

**对比类页面**：
```
"{topic} comparison analysis"
"{topic} pros cons evaluation"
"{主题} 对比分析" / "{主题} 优劣势"
```

### 1.4 时间限定

- 默认添加当前年份或前一年
- 趋势/预测类查询：添加 "2025-2026" 或 "latest"
- 经典案例类查询：可放宽至 3 年内
- 基础技术类查询：不限时间

### 1.5 可信度增强关键词

在查询中酌情添加：
- `report`、`whitepaper`、`研究报告`、`白皮书`
- `Gartner`、`McKinsey`、`IDC`、`Forrester`
- `statistics`、`data`、`survey`
- `PDF`（倾向于获取正式报告）

## 2. 来源评估框架

### 2.1 来源可信度评分（六档量化体系）

| 分值 | 等级 | 来源类型 | 示例 |
|:----:|:----:|----------|------|
| 90-100 | A+ | 权威机构（政府、国际组织） | WHO, WIPO, 国家统计局, UN机构 |
| 80-89 | A | 企业官方（年报、财报） | OpenAI 年报, 头部企业财报, 上市公司公告 |
| 70-79 | A- | 学术论文 | Nature, Science, arXiv, IEEE, ACL |
| 65-69 | B+ | 权威媒体 | MIT Tech Review, Bloomberg, Reuters, FT |
| 60-64 | B | 行业媒体 | TechCrunch, 36氪, 机器之心, InfoQ |
| <60 | C | 自媒体/内容农场（应排除） | 无明确来源转载, 营销软文, 标题党 |

**评分原则**：
- 分值代表来源的可信度等级区间，边界分数归入较高等级（如 80 分归入 A 而非 A-）
- 同一来源按内容类型可能浮动：如 Gartner 官方报告可达 90+，但博客文章可能仅 65+
- **低于 60 分的来源一律排除**，不进入研究素材

### 2.1.1 幽灵来源识别

幽灵来源是高风险内容特征，需特别警惕并标记：

**高风险特征**：
- 无 URL、DOI 与出版信息不匹配
- 标题/年份与内容矛盾
- 无法回溯的二手转述（"据知情人士透露"、"消息人士称"）
- 引用来源与内容领域不符（如 AI 论文引用音乐网站）
- 发布时间异常（未来日期或极旧日期）

**处理流程**：
1. **标记**：识别幽灵来源特征，记录问题
2. **补充**：尝试补充可验证信息（搜索原发布渠道、查找 DOI/URL）
3. **降级/移除**：无法核实的降级为低可信度（<60）并排除

### 2.1.2 来源筛选标准

**必选条件**（至少满足一条）：
- 来源可信度 ≥ 60
- 包含第一手数据或原始研究
- 来自权威机构的正式发布

**加分条件**：
- 发布时间在 1 年以内
- 引用了其他权威来源
- 包含图表或结构化数据
- 提供了方法论说明

**排除条件**（满足任一即排除）：
- 纯观点无数据支撑
- 来源不明的二手转述
- 明显的商业推广内容
- 信息与其他多个来源矛盾且无法解释
- 可信度 < 60

## 3. 信息提炼方法

### 3.1 提炼原则

- **精确性**：提取精确数据，不做模糊化处理
- **可溯源**：每条信息标注来源 URL
- **去重**：多个来源报道同一事实时，选择最权威/最近的
- **交叉验证**：关键数据点尽量有 2+ 来源确认

### 3.2 提炼模板

对每个抓取的页面，按以下维度提炼：

```
1. 核心论断：这篇文章/报告的主要结论是什么？
2. 关键数据：有哪些具体数字、统计、比例？
3. 趋势判断：对未来有什么预测或趋势判断？
4. 案例信息：提到了哪些具体公司、产品、项目？
5. 引用价值：哪些信息可以直接用于 PPT 页面？
```

### 3.3 数据格式化

| 数据类型 | 格式化方式 | 示例 |
|----------|-----------|------|
| 市场规模 | 金额 + 货币 + 年份 | "2025年全球AI Agent市场规模达1240亿美元" |
| 增长率 | 百分比 + 时间段 | "年复合增长率（CAGR）达34.2%（2024-2028）" |
| 占比 | 百分比 + 分母 | "企业级应用占整体市场的67%" |
| 排名 | 序号 + 指标 | "中国在AI专利数量上排名全球第二" |
| 时间点 | 完整日期 + 事件 | "2025年3月，OpenAI发布Agents SDK" |

## 4. 交叉验证策略

### 4.1 验证级别

| 级别 | 要求 | 适用场景 |
|------|------|----------|
| 强验证 | 3+ 独立来源确认 | 关键数据点、市场规模、核心结论 |
| 标准验证 | 2 个来源确认 | 趋势判断、技术描述 |
| 弱验证 | 1 个权威来源 | 案例描述、观点引用 |

### 4.2 矛盾处理

当不同来源提供矛盾信息时：

1. **标注矛盾**：在研究报告中明确标注不同来源的不同说法
2. **优先级**：权威来源 > 时间更近 > 引用更多
3. **呈现方式**：
   - "据 Gartner 预测市场规模为 X 亿美元 [1]，而 IDC 的估计为 Y 亿美元 [2]"
   - 不隐藏矛盾，让下游使用者自行判断

## 5. 研究深度配置

### 5.1 L1 快速研究

| 配置项 | 值 |
|--------|-----|
| 搜索轮数 | 2-4 轮 |
| 总来源要求 | ≥10 个 |
| 平均可信度 | >60/100 |
| 交叉验证 | 弱验证（标注单源数据） |
| 字数要求 | ≥3,000 字 |
| 适用场景 | 了解基本面、时间紧迫 |
| 预期耗时 | 3-5 分钟 |

### 5.2 L2 标准研究

| 配置项 | 值 |
|--------|-----|
| 搜索轮数 | 5-8 轮 |
| 总来源要求 | ≥20 个 |
| 平均可信度 | >70/100 |
| 交叉验证 | 标准验证（2+ 独立来源） |
| 字数要求 | ≥5,000 字 |
| 适用场景 | 行业分析、多数使用场景 |
| 预期耗时 | 5-15 分钟 |

### 5.3 L3 专家级研究

| 配置项 | 值 |
|--------|-----|
| 搜索轮数 | 8-12 轮 |
| 总来源要求 | ≥30 个 |
| 平均可信度 | >70/100 |
| 交叉验证 | 强验证（3+ 独立来源） |
| 字数要求 | ≥8,000 字 |
| 幽灵来源检查 | 必须执行 |
| 适用场景 | 决策支持、重要分析 |
| 预期耗时 | 15-30 分钟 |
